PROJECT OVERVIEW
================
This package contains the R workflow and data used to generate 30mx30m suitability model (HSM) raster layer for 1985, 2000, and 2020, covering a large portions of Ontario and Quebec. The creation of these models is based on HSM methods from Leblond et al. 2014. It uses publicly available, national-scale, historical, rasterized datasets for forest attributes (SCANFI) and disturbance type/year (CanLAD, NFDB), and mines (MinCan) to build the Leblond et al. (2014) model at the three time points of interest. 
Specifically, this workflow: 
	1) Generates intermediate habitat and mine layers;
	2) Uses these layers to create habitat suitability models (HSMs) at years 1985, 2000, and 2020,
	based on caribou suitability model parameters and equations presented in Leblond et al. 2014;
	3) Generates 'change in HSI' products for 1985-2020, 1985-2000, and 2000-2020;
	4) Generates results figures, including maps and a PCoA analysis of change in habitat composition over time for Quebec and Ontario 	boreal caribou herds. 

The project has been packaged so that the folder can be placed anywhere on a computer and used as the project root. All R scripts are written to detect the root directory automatically and use relative paths from that location.

The workflow covers:
- habitat raster preprocessing and classification
- mine preprocessing and mine-distance / mine-density layers
- final HSI model calculation
- masking and HSI change layers
- map figures
- herd-level extraction tables
- herd spatial classification relative to the timber allocation line
- PCoA ordination figure
- mean HSI by latitude summaries

IMPORTANT GENERAL NOTES ON R SCRIPTS 
====================================
1. To preserve folder paths when running, open the folder "caribou_hindcast_packaged" as the project working directory.
   Do not run the scripts from inside the R/ folder alone.

2. Most scripts contain a "User inputs" section near the top. This is where the user selects:
   - which year(s) to run (1985, 2000, and/or 2020)
   - input / output options
   - tuning parameters where applicable

3. Many outputs are written automatically into subfolders if those folders do not already exist.

4. Some scripts create large temporary raster files. Ensure there is enough disk space available, especially in the project temp directory.

5. The road rasters used by the modelling scripts are treated as existing project data in data/roads/. They are required inputs but are not generated by the R scripts listed here. More on this in the 'DATA' section, below. 

6. The 'stand age' layers for 1985, 2000, and 2020 that are used in the habitat classification portion of this workflow are created in a separate packaged R workflow. More on this in the 'DATA' section, below. 


DATA
======	
This package includes all raw input data required to run the workflow and reproduce
the results generated by the scripts in this directory. In this context, "raw" data
means input files that were not created by the workflow itself, even if some of them
were pre-processed externally before being placed in the package.

All raw data used to generate the maps, tables, and outputs in this package are stored
in the appropriate project subfolders. The workflow then reads these files and creates
the intermediate and final outputs used throughout the analysis.

Users who only want to reproduce the packaged workflow should use the data already
included in this project directory.

Users who want to rerun the workflow using newer source versions may replace the raw
data files with updated versions, provided that:
- file names and folder locations are adjusted as needed in the relevant scripts, and
- the updated datasets remain compatible with the workflow assumptions.

A summary of the major raw data used in this package is outlined below:

DIRECTORY STRUCTURE
===================
The project root is expected to contain at least the following folders:

- R/
  R scripts. See section R SCIPTS INCLUDED, below, for more information. 

- data/
  Main project data directory, contains all data needed for the workflow. 
  Data is split by workflow section (e.g., 'habitat', 'roads', 'mines', 'postprocessing'), each with subfolders distinguishing between 'raw' and           
  'processed' (output) data.

- HSI_output/
  Final HSI rasters, masked HSI rasters, and dHSI rasters. Used for final model outputs. 

- map_data/
  Supporting map layers for visualization and spatial analysis, including timber allocation limits.

- map_layouts/
  Map figure outputs.

- plots/
  Plot figure outputs (PCoA).

- tables/
  Tabular outputs used by later post-processing steps.

- temp/
  Temporary raster processing directory used by some scripts.

PORTABILITY
===========
This package was refactored so that file paths are relative to the project root.
In general, the user should:

1. download or copy the full caribou_hindcast_packaged folder
2. keep the internal folder structure unchanged
3. open that folder as the working directory or project root
4. run scripts from the R/ folder in workflow order

If automatic root detection fails in a given session, manually set the working
directory to the project root before running the script.


TROUBLESHOOTING
===============
1. Missing file errors
   Check that:
   - the project root is correct
   - the internal folder structure has not changed
   - all required raw and intermediate files exist
   - earlier scripts in the workflow have already been run

2. Raster alignment errors
   These usually mean one or more upstream rasters do not match the expected grid,
   extent, or projection. Recheck the earlier preprocessing steps.

3. Large raster processing / memory issues
   Try:
   - running fewer years at once
   - increasing available disk space (put this package somewhere with a lot of space)
   - reducing chunk size or aggregation settings where provided (keep in mind this will likely increase run time)
   - rerunning from the last completed step

4. Run time efficiency
   This workflow processes and creates very large, high resolution raster layers. There are built in systems to maximize run time efficiency,  
   but the entire workflow will take weeks to run. Adjust parallel/chunk processing from the 'user inputs' section at the beginning of each 
   script. 

4. Download-related issues
   Some scripts check for local files first and may try to download missing files.
   If automatic download fails, save the required file manually into the folder
   specified in the script.
